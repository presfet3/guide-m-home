# guide-m-home

这是一个用于归档和发布多个独立 HTML 页面的仓库。仓库本身不面向任何特定网站或域名，仅作为静态页面集合的托管与分发空间。

## 项目简介

本仓库的主要目的是集中存放一些独立的、功能单一或信息展示型的 HTML 页面。这些页面彼此独立，不构成完整的网站结构，适合作为临时页面、工具页面或文档页面的归档。

仓库采用 GitHub Pages 进行页面托管，所有 HTML 文件均存放于根目录下，方便直接访问与分享。

## 目录说明

仓库根目录直接存放各个 HTML 页面文件，不设置子目录结构（除非页面需要关联资源）。每个 HTML 文件均为一个独立可访问的页面。

```
/
├── index.html          # 仓库入口页面（可选）
├── page-example.html   # 示例页面
└── ...
```

- `index.html` 为仓库入口页面，用于简要展示所有归档页面的列表或说明。
- 其他 `.html` 文件为具体的归档页面，文件名即页面标识。

## 页面归档说明

所有页面均为纯前端 HTML，不依赖后端服务。页面内容可能包括：

- 信息展示页
- 工具型页面
- 临时说明页
- 其他独立用途的页面

页面之间没有逻辑关联，每个页面均可单独访问。

## 维护说明

- 新增页面：直接向仓库根目录提交新的 `.html` 文件。
- 更新页面：修改对应文件后提交更新。
- 删除页面：移除不再需要的文件，并确保 `index.html` 中的链接同步更新。
- 本仓库不提供后端接口或数据库支持，所有数据均内置于 HTML 文件中。

## 使用方式

通过 GitHub Pages 访问仓库，直接浏览根目录下的 HTML 文件即可。例如：

```
https://<username>.github.io/guide-m-home/page-name.html
```

（请将 `<username>` 替换为实际的 GitHub 用户名。）

## 注意事项

- 本仓库为纯归档用途，不保证页面内容的长期可用性。
- 页面内容可能随时更新或移除，恕不另行通知。
- 仓库不接受外部贡献或 issue 反馈，请勿提交无关内容。
